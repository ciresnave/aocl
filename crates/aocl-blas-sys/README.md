# aocl-blas-sys

Raw `unsafe extern "C"` bindings to AOCL-BLAS (BLIS), generated by `bindgen` at build time.

For a safe, idiomatic API on top of these bindings see [`aocl-blas`](../aocl-blas/).

## Build requirements

- An AOCL install. Set `AOCL_ROOT` if it is not at the OS-default location.
- `libclang` (used by `bindgen`). On Windows, install LLVM and ensure `libclang.dll` is at `C:\Program Files\LLVM\bin\` or set `LIBCLANG_PATH` explicitly.

## Cargo features

| Feature | Effect |
|---------|--------|
| `ilp64`       | Use 64-bit integer indexing (`ILP64/`) instead of the default 32-bit (`LP64/`). |
| `static-link` | Link against the static AOCL libraries instead of the DLL imports. |
