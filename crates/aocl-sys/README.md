# aocl-sys

Raw `unsafe extern "C"` FFI bindings to the [AMD Optimizing CPU Libraries (AOCL)](https://www.amd.com/en/developer/aocl.html), generated by `bindgen` against the AOCL C headers at build time.

This crate is the low-level layer; for safe, idiomatic Rust APIs use the [`aocl`](../aocl/) crate.

## Build requirements

- An AOCL installation. Set `AOCL_ROOT` to the install root if it is not at the OS-default location:
  - Windows: `C:\Program Files\AMD\AOCL-Windows`
  - Linux:   `/opt/AMD/aocl/aocl-linux-*`
- `libclang` for `bindgen`. On Windows, install LLVM and (if needed) set `LIBCLANG_PATH`.

## Cargo features

One feature per AOCL component, plus build-mode toggles. See the workspace [README](../../README.md).

## Layout

Each enabled feature exposes a top-level Rust module under `aocl_sys::`:

```rust
#[cfg(feature = "utils")]
use aocl_sys::utils;
```

The contents of each module are produced by `bindgen` from the corresponding header in `wrapper/`.
